# Полный пошаговый гайд: сборка строб-системы с ESP32

Цель: собрать систему, где Android-приложение управляет ESP32 по BLE, а ESP32 включает внешние драйверы света. ESP32 не питает лампы и не коммутирует 12V напрямую.

## 0. Главные правила

- 12V нельзя подавать на GPIO ESP32.
- ESP32 питается только через нормальный DC-DC 12V -> 5V.
- Нагрузка подключается только через MOSFET/реле/драйвер.
- Минус ESP32 и минус силовой части должны быть соединены.
- Перед пайкой питание отключено.
- Первый запуск делается на столе, не в машине.
- Сначала тестовая лампа/нагрузка, потом реальная система.

## 1. Что купить

Минимум:

- ESP32 DevKit V1 или похожая плата ESP32.
- DC-DC преобразователь 12V -> 5V, минимум 1A, лучше 2A.
- 8-канальный MOSFET-модуль для логического управления или 8 отдельных logic-level MOSFET-каналов.
- Предохранитель на линию 12V.
- Держатель предохранителя.
- Провода подходящего сечения под ток нагрузки.
- Термоусадка.
- Клеммники или нормальные автомобильные разъемы.
- Мультиметр.
- Паяльник, припой, флюс.

Рекомендуется:

- TVS-диод на 12V питание.
- Отдельный выключатель питания системы.
- Блок питания с ограничением тока для первого теста.
- Резисторы 100-220 Ом в затвор MOSFET, если собираешь MOSFET-канал сам.
- Резисторы 10 кОм gate-to-GND, если собираешь MOSFET-канал сам.

## 2. Какие каналы есть в прошивке

По умолчанию прошивка использует 8 каналов:

| Канал в приложении | GPIO ESP32 | Назначение |
|---|---:|---|
| `FrontLeft` | GPIO16 | Передний левый |
| `FrontRight` | GPIO17 | Передний правый |
| `RearLeft` | GPIO18 | Задний левый |
| `RearRight` | GPIO19 | Задний правый |
| `SideLeft` | GPIO21 | Боковой левый |
| `SideRight` | GPIO22 | Боковой правый |
| `Beacon` | GPIO23 | Маяк |
| `Flood` | GPIO25 | Заливной свет |

Эта таблица задается в `firmware/esp32_ble_controller/Config.h`.

## 3. Правильная схема подключения одного канала

Вариант с готовым MOSFET-модулем:

```text
ESP32 GPIO16  -> IN1 MOSFET-модуля
ESP32 GND     -> GND MOSFET-модуля
12V +         -> + нагрузки
- нагрузки    -> OUT-/DRAIN MOSFET-модуля
GND 12V       -> GND/SOURCE MOSFET-модуля
```

Вариант с обычным реле-модулем:

```text
ESP32 GPIO16  -> IN1 реле-модуля
ESP32 GND     -> GND реле-модуля
5V            -> VCC реле-модуля, если модуль 5V
12V +         -> COM реле
NO реле       -> + нагрузки
- нагрузки    -> GND 12V
```

Для стробов и светодиодных модулей лучше MOSFET, а не механическое реле. Реле медленнее и изнашивается на частом мигании.

## 4. Как паять

1. Нарисуй таблицу: какой свет к какому каналу.
2. Отрежь провода с запасом по длине.
3. Зачисти 4-6 мм изоляции.
4. Залуди конец провода.
5. Залуди площадку/контакт.
6. Припаяй провод, не перегревая плату.
7. Проверь, что нет перемычки между соседними контактами.
8. Закрой соединение термоусадкой.
9. Проверь мультиметром прозвонку и отсутствие короткого замыкания между `12V` и `GND`.

Не делай скрутки без пайки/обжима. В машине от вибрации они быстро становятся проблемой.

## 5. Питание

Подключение питания:

```text
Аккумулятор/12V источник
-> предохранитель
-> выключатель, если ставишь
-> DC-DC 12V -> 5V
-> 5V pin ESP32 или USB ESP32
```

Перед подключением ESP32:

1. Включи DC-DC без ESP32.
2. Мультиметром проверь выход.
3. Должно быть около `5.0V`.
4. Если там 12V или больше 5.3V, ESP32 не подключать.

## 6. Настройка прошивки

Открой:

```text
firmware/esp32_ble_controller/Config.h
```

Пины:

```cpp
constexpr uint8_t kChannelPins[kChannelCount] = {16, 17, 18, 19, 21, 22, 23, 25};
```

Инверсия:

```cpp
constexpr bool kChannelInverted[kChannelCount] = {
    false, false, false, false, false, false, false, false};
```

Если драйвер включается уровнем HIGH, оставь `false`.

Если драйвер включается уровнем LOW, поставь `true` для нужного канала.

Пример: `Beacon` на GPIO23 активен по LOW:

```cpp
constexpr bool kChannelInverted[kChannelCount] = {
    false, false, false, false, false, false, true, false};
```

Важно: поле `Invert Signal` в приложении сейчас не меняет прошивку. Реальная инверсия задается только в `Config.h`.

## 7. Прошивка ESP32

1. Установи Arduino IDE.
2. Открой `File -> Preferences`.
3. В `Additional Boards Manager URLs` добавь:

```text
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
```

4. Открой `Tools -> Board -> Boards Manager`.
5. Установи `ESP32 by Espressif Systems`.
6. Открой файл:

```text
firmware/esp32_ble_controller/esp32_ble_controller.ino
```

7. Выбери плату `ESP32 Dev Module`.
8. Выбери COM-порт.
9. Нажми `Upload`.
10. Открой `Serial Monitor`.
11. Скорость: `115200`.
12. Должно быть `READY`.

## 8. Установка приложения

1. Установи APK:

```text
dist/StagePatch-release.apk
```

2. Включи Bluetooth.
3. Выдай разрешения Bluetooth и Location.
4. Открой вкладку `Connect`.
5. Нажми `Scan Controllers`.
6. Выбери только `ESP32-StrobeCtrl`.
7. Нажми `Connect`.

## 9. Что писать в Devices

Вкладка `Devices` нужна как профиль и список твоих подключений.

Пример для `FrontLeft`:

| Поле | Что писать |
|---|---|
| `Name` | `Front Left` |
| `Type` | `front` |
| `Channel` | `1` |
| `Primary Output` | `16` |
| `Group` | `FRONT` |
| `Mode` | `strobe` |
| `Brightness` | `255` |
| `Invert Signal` | только как заметка, реальная инверсия в `Config.h` |

Пример для `Beacon`:

| Поле | Что писать |
|---|---|
| `Name` | `Beacon` |
| `Type` | `beacon` |
| `Channel` | `7` |
| `Primary Output` | `23` |
| `Group` | `BEACON` |
| `Mode` | `steady` |
| `Brightness` | `255` |

Важно: ручные кнопки `Control` управляют фиксированными каналами прошивки. `Devices` пока не переназначает GPIO автоматически.

## 10. Первый тест на столе

1. Подключи только ESP32 и один драйвер.
2. Вместо реального света подключи тестовую лампу или простую безопасную нагрузку.
3. Подай питание через БП с ограничением тока, если он есть.
4. Подключись в приложении.
5. Нажми `Front Left`.
6. Нагрузка должна включиться.
7. Нажми `Front Left` еще раз.
8. Нагрузка должна выключиться.
9. Нажми `All Off`.
10. Все должно быть выключено.

Если канал включен сразу после старта, не подключай реальную нагрузку. Проверь `kChannelInverted`.

## 11. Тест fail-safe

1. Включи один канал.
2. Выключи Bluetooth на телефоне.
3. Каналы должны выключиться после fail-safe.
4. Включи Bluetooth обратно.
5. Подключись заново.
6. Проверь `All Off`.

## 12. Команды, которые понимает прошивка

Базовые:

```text
PING
STATUS
STOP
ALL_OFF
```

Каналы:

```text
FrontLeft=ON
FrontLeft=OFF
Beacon=ON
Beacon=OFF
```

Группы:

```text
MODE=ON;GROUP=FRONT
MODE=OFF;GROUP=REAR
MODE=ON;GROUP=SIDE
```

Строб:

```text
MODE=STROBE;CH=FrontLeft,RearRight;ON=80;OFF=80;REP=5;PAUSE=300
```

## 13. Перед установкой в машину

Проверить:

- На выходе DC-DC ровно около 5V.
- Между 12V и GND нет короткого.
- ESP32 GND соединен с GND драйверов.
- Каждый канал выключен после старта ESP32.
- `All Off` выключает каждый канал.
- При потере Bluetooth выходы выключаются.
- Провода закреплены и не перетираются.
- Все пайки закрыты термоусадкой.
- Предохранитель стоит на 12V линии.

## 14. Что не делать

- Не подключать ESP32 к штатным важным цепям автомобиля.
- Не лезть в airbag, ABS, ECU, CAN, штатные жгуты безопасности.
- Не оставлять голые контакты.
- Не ставить предохранитель большего номинала, чем выдерживает проводка.
- Не тестировать строб на дороге без понимания местных правил.

## 15. Если что-то не работает

Порядок диагностики:

1. Serial Monitor показывает `READY`.
2. Телефон видит `ESP32-StrobeCtrl`.
3. Приложение подключается.
4. GPIO меняет состояние при команде.
5. На вход драйвера приходит сигнал.
6. У драйвера есть питание.
7. Есть общий GND.
8. Нагрузка подключена через силовую часть.
9. Инверсия в `Config.h` соответствует модулю.

Если есть сомнение, отключи нагрузку и вернись к тесту одного канала.
