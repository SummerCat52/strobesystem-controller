# Пошаговый гайд: StagePatch + ESP32-StrobeCtrl

Цель: телефон управляет ESP32 по BLE, ESP32 включает только логические входы драйверов, а силовая часть отдельно коммутирует автомобильные 12V. Главный приоритет: не подавать 12V на ESP32 и иметь предсказуемое выключение всех каналов.

## 1. Что отправлять другу

- Полный пакет: `dist/StagePatch_friend_package.zip`
- Только Android-приложение: `dist/StagePatch-release.apk`
- Только прошивка ESP32: `dist/ESP32_firmware_only.zip`

## 2. Что уже зашито в прошивке

- BLE имя: `ESP32-StrobeCtrl`
- Service UUID: `5E7A1001-0000-4C0A-B001-112233445566`
- Command UUID (write): `5E7A1002-0000-4C0A-B001-112233445566`
- Status UUID (read/notify): `5E7A1003-0000-4C0A-B001-112233445566`

Каналы по умолчанию:

- `FrontLeft` -> GPIO16
- `FrontRight` -> GPIO17
- `RearLeft` -> GPIO18
- `RearRight` -> GPIO19
- `SideLeft` -> GPIO21
- `SideRight` -> GPIO22
- `Beacon` -> GPIO23
- `Flood` -> GPIO25

Пины и реальная инверсия выходов задаются в `firmware/esp32_ble_controller/Config.h`.

## 3. Критичные запреты

- Нельзя подключать 12V напрямую к GPIO ESP32.
- Нельзя питать ESP32 напрямую от автомобильных 12V.
- Нельзя коммутировать лампы, строб-модули, реле или другую нагрузку через GPIO.
- Нельзя делать монтаж при включенном питании.
- Нельзя проверять первый запуск сразу на штатной дорогой нагрузке.

## 4. Правильная электрическая логика

ESP32 должна управлять только входом драйвера:

1. `GPIO ESP32` -> `IN драйвера/MOSFET-модуля/реле-модуля/оптопары`.
2. `GND ESP32` -> `GND драйвера`.
3. `12V нагрузка` -> через силовой драйвер, не через ESP32.
4. `12V авто` -> предохранитель -> DC-DC -> ESP32.

Для автомобиля обязательно:

- предохранитель на 12V линии ближе к источнику питания;
- DC-DC преобразователь automotive-класса или с запасом по помехам;
- общий `GND` ESP32 и драйверов;
- TVS-диод/защита от выбросов по питанию;
- диод подавления выбросов для индуктивных нагрузок;
- нормальное сечение проводов под ток нагрузки.

## 5. Важное про Inverted

В текущем релизе поле `Inverted` в приложении не меняет электрическую инверсию прошивки. Реальная инверсия выходов задаётся в `Config.h`:

```cpp
constexpr bool kChannelInverted[kChannelCount] = {
    false, false, false, false, false, false, false, false};
```

Если модуль реле или драйвер включается уровнем LOW, для нужного канала поставь `true` и перепрошей ESP32.

Пример: если `Beacon` на GPIO23 активен по LOW, седьмой элемент должен быть `true`:

```cpp
constexpr bool kChannelInverted[kChannelCount] = {
    false, false, false, false, false, false, true, false};
```

## 6. Первый запуск без риска

1. Отключи силовую нагрузку.
2. Прошей ESP32.
3. Открой `Serial Monitor` на `115200`.
4. Проверь, что после старта есть `READY`.
5. Подключи один тестовый канал через драйвер и простую тестовую нагрузку.
6. Используй блок питания с ограничением тока, если он есть.
7. Проверь один канал: ON -> OFF -> `ALL_OFF`.
8. Только после этого переходи к остальным каналам.

## 7. Прошивка ESP32

1. Установи Arduino IDE.
2. Установи пакет `ESP32 by Espressif Systems`.
3. Открой `firmware/esp32_ble_controller/esp32_ble_controller.ino`.
4. В `Tools` выбери `ESP32 Dev Module`.
5. Выбери правильный COM-порт.
6. Нажми `Upload`.
7. В `Serial Monitor` проверь `READY`.

## 8. Установка APK

1. Установи `dist/StagePatch-release.apk` на Android.
2. Включи Bluetooth.
3. Выдай приложению разрешения Bluetooth и Location.

## 9. Подключение к контроллеру

1. Открой вкладку `Connect`.
2. Нажми `Scan Controllers`.
3. Выбери только `ESP32-StrobeCtrl`.
4. Нажми `Connect`.

Не подключайся к телевизорам, наушникам и другим BLE-устройствам.

## 10. Как заполнять Devices

`Devices` -> `Add Device`. Данные берутся из фактической проводки:

- `Name`: понятное имя, например `Front Left`.
- `Type`: роль света, например `front`, `rear`, `beacon`, `flood`.
- `Channel`: логический канал управления.
- `Primary Output`: физический выход ESP32 или номер канала драйвера.
- `Group`: группа, например `FRONT`, `REAR`, `SIDE`.
- `Brightness`, `Mode`: справочные параметры текущего профиля.

Важно: в текущем релизе настройка `Devices` помогает вести профиль, но ручные кнопки `Control` управляют фиксированными каналами прошивки (`FrontLeft`, `FrontRight`, и т.д.).

## 11. Безопасный функциональный тест

1. В `Control` нажми `Front Left` -> тестовый выход должен включиться.
2. Нажми `Front Left` повторно -> выход должен выключиться.
3. Нажми `All Off` -> все каналы должны выключиться.
4. Запусти паттерн и проверь `All Off`.
5. Отключи Bluetooth на телефоне: каналы должны выключиться после fail-safe.
6. Перезагрузи ESP32: после старта все каналы должны быть OFF.

## 12. Что чаще всего сжигает железо

- 12V попали на GPIO ESP32.
- Нет общего `GND` между ESP32 и драйвером.
- Перепутана полярность питания.
- Нет предохранителя на 12V линии.
- Реле/катушка без защитного диода.
- Active-low модуль не настроен через `Config.h`.
- Первый тест сделан сразу на реальной нагрузке без ограничения тока.
- Использованы опасные boot/strapping GPIO при самостоятельной смене пинов.

## 13. Если меняешь GPIO

Используй обычные GPIO, которые не влияют на загрузку ESP32. Не выбирай boot/strapping пины без проверки документации конкретной платы. После смены пинов проверь старт ESP32 с полностью отключенной нагрузкой.

## 14. Диагностика

Проверяй по порядку:

1. В Serial есть `READY`.
2. Телефон видит `ESP32-StrobeCtrl`.
3. Разрешения Bluetooth/Location выданы.
4. Есть общий `GND`.
5. Нагрузка подключена через драйвер.
6. Инверсия active-low каналов задана в `Config.h`.
7. `All Off` реально выключает каждый канал.
