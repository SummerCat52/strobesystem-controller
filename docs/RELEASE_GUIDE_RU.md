# Как выпускать APK через GitHub Release

Этот проект уже настроен так, чтобы GitHub сам собирал APK и прикреплял его к релизу.

## Где текущий APK

Текущий релиз:

https://github.com/SummerCat52/strobesystem-controller/releases/tag/v0.1.0

Файл для установки на телефон:

- `StagePatch-release.apk`

Файлы для друга/сборщика:

- `StagePatch-release.apk` - приложение на Android.
- `ESP32_firmware_only.zip` - только прошивка ESP32.
- `StagePatch_sources.zip` - исходники приложения, прошивки, документы и примеры.

## Как выпустить новую версию

1. Убедись, что все изменения запушены в `main`.
2. Придумай номер версии, например `v0.1.1`.
3. В PowerShell из папки проекта выполни:

```powershell
git tag v0.1.1
git push origin v0.1.1
```

4. GitHub Actions автоматически запустит сборку.
5. После успешной сборки появится новый Release с APK.

## Как выпустить APK без тега

1. Открой репозиторий на GitHub.
2. Перейди во вкладку **Actions**.
3. Выбери workflow **Build APK and publish release**.
4. Нажми **Run workflow**.
5. В поле `tag` можно написать, например `v0.1.1-test`.
6. GitHub соберет APK и прикрепит его к релизу.

## Что проверяет GitHub перед APK

Перед публикацией GitHub делает:

- `flutter pub get`
- `flutter analyze`
- `flutter test --coverage`
- `flutter build apk --release`

Если что-то падает, APK не публикуется. Это хорошо: сломанная версия не уйдет другу случайно.

## Что отправлять другу

Если другу нужно только установить приложение:

- дай ссылку на Release;
- он скачивает `StagePatch-release.apk`;
- устанавливает APK на Android;
- BLE-подключение делает внутри приложения, не через системное меню Bluetooth.

Если другу нужно прошивать ESP32:

- дай ссылку на Release;
- он скачивает `ESP32_firmware_only.zip`;
- открывает `.ino` в Arduino IDE;
- прошивает ESP32 по гайду из архива.

Если другу нужны все исходники:

- дай ссылку на Release;
- он скачивает `StagePatch_sources.zip`.

## Важно по безопасности

- Перед подключением к машине сначала проверить на LED или мультиметре.
- ESP32 GPIO не должны питать нагрузку напрямую.
- Для автомобильной нагрузки нужны MOSFET/реле/драйверы, предохранитель, общий GND и защита питания.
- Если приложение потеряло связь, прошивка должна выключить выходы через fail-safe.
